Kickstarter offers all information via an MQTT broker. The application within the Kickstarter container sends all information to an MQTT broker. This web site connects to that MQTT broker and visualizes the settings and the progress of getting new devices configured. So both "MQTT broker" and "Kickstarter" have to be "online" for everythin to work.

In case Kickstarter has been configured to upgrade the firmware, the target version is displayed here, too.

Kickstarter can keep track of the devices it processed in the last phase of processing a device, which is called "aftercare". A new file can get started with the "reset" button.

The container runs a router advertiser daemon, that announces a local IPv6 network. In default settings this is fdc2::/24.

All IP devices connected to this LAN can now be contacted, even if they are in default settings and use the same IPv4 address 192.168.1.1/24.

The settings site should update itself in real time, a reload is normally not necessary.
